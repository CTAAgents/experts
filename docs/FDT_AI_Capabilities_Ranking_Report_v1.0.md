## 4. 八大维度排名分析

### 4.1 Agent 协作深度

**排名: 第 1 名 / 11 系统**

| 系统 | Agent 数 | 角色分工 | 并行粒度 | 调度架构 | 评分 |
|:-----|:--------:|:---------|:---------|:---------|:---:|
| **FDT** | **9** | **9 独立角色,边界钉死** | P3三源+P4两源并行 | 明鉴秋统一调度 | **S** |
| TradingAgents | 7 | 分析师/研究员/交易员/风控 | 分析师并行 | 基金经理调度 | A |
| AI Berkshire | 4-8 | 4大师观点+spawn团队 | 并行spawn | TeamLead调度 | A |
| CSTrader | 8 | 5推理+3操作 | 信号聚合 | 无严格编排 | B+ |
| FinSight | 7 | 7金融专家 | 并行 | 协调器 | B+ |
| AgenticAITA | 3 | 分析师→风控→交易员 | 串行 | 管线 | B |
| AlphaCrafter | 3 | Miner→Screener→Trader | 串行 | 管线 | B |
| Macro Economists | 4 | Hawk/Dove/Debate/Rule | 混合 | 无 | B |
| Vibe-Trading | 1 | 通用 | 无 | 无 | C |
| OpenBB | 1 | 数据查询 | 无 | 无 | C |

**关键差异**:
- FDT 是全网唯一明确定义 **每个 Agent "做什么"和"不做什么"** 的系统
- FDT 的 P3 按需并行数据源设计（闫判官智能调度所需源）业界独有
- 9 Agent 的角色不重叠、不越界，构成完整的辩论制衡体系

### 4.2 辩论机制

**排名: 第 1 名 / 11 系统**

| 系统 | 辩论结构 | 分歧处理 | 裁决机制 | 评分 |
|:-----|:---------|:---------|:---------|:---:|
| **FDT** | **P4并行辩论→P5串行裁决** | **分歧收敛条件边** | **闫判官含交易参数→风控明** | **S** |
| TradingAgents | 多空辩论+协调者 | 协调者选观点 | 基金经理执行 | A |
| AgenticAITA | 串行审议链 | 硬门过滤 | 执行Agent | B+ |
| CSTrader | 信号聚合 | 无 | 无 | B |
| Macro Economists | 辩论Agent | 无 | 无固定结构 | B- |
| AI Berkshire | 观点冲突 | 无 | 用户决策 | B- |

**关键差异**:
- FDT 是唯一同时支持 **并行辩论 + 串行裁决 + 条件分歧路由** 的系统
- 来自 **Macro Economists 论文的核心发现**（Delta Sharpe -0.004, p=0.769）：平等投票式辩论不优于单 Agent，但 FDT 的 **非平等辩论结构**（多空举证→闫判官裁决）避开了这个陷阱
- FDT 的风控明直接基于闫判官 verdict 审核，而非参与方向判断，确保角色独立性

### 4.3 自进化闭环

**排名: 第 1 名 / 11 系统**

| 系统 | 自进化步骤 | 触发条件 | 覆盖范围 | 生产运行 | 评分 |
|:-----|:----------|:---------|:---------|:--------:|:---:|
| **FDT** | **验证→校准→进化→ML→Skill** | ≥5/≥50样本+冷却 | **全链路** | **是** | **S** |
| AlphaCrafter | 因子挖掘循环 | 持续 | 因子层面 | 否 | B+ |

| 其他 7 系统 | 无 | - | - | - | D |

**绝对领先**:
- FDT 是全网 **唯一** 在生产环境中运行完整自进化闭环的交易系统
- 进化由 **实际行情验证驱动**，无需人工标注
- 覆盖从指标验证（validate_verdicts.py）到机器学习（ml/trainer.py）的全范围
- 额外包含 SkillAdaptor（故障归因）+ Skillevolver（技能层进化）+ Autoresearch（A/B 验证）三层技能进化
