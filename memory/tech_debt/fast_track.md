# 单品种快车道约定（v1.0 · 2026-07-07 确立）

> 环境优化，不改变核心 SOP 逻辑。由团队主管(明鉴秋)在单品种(模式三)/批量(模式二)模式应用。

## 一、提速根因（LH2609 首跑实测）
1. **venv 未初始化**：专家团 venv 原为空白，每个 Agent 首次运行现装 pandas/numpy 等，数技源单次装包耗时 ~18 分钟。→ 已冻结 `requirements.lock`，新环境 `pip install -r requirements.lock`。
2. **Agent 卡死/极慢**：观澜(技术面)两次卡在 K 线取数网络等待(~8分钟)，闫判官/风控明各耗时 3-5 分钟（LLM 大上下文推理）。→ 引入看门狗 + 主管兜底。
3. **重复 I/O**：各 Agent 各自拉 K 线/基本面。→ 数据一次拉取后文件共享（SOP 已如此）。

## 二、快车道规则
- **单品种模式**：只扫指定品种（如 `scan_all.py --symbols LH2609`），**不跑 62 全扫描、不做同链 30+ 品种去重统计**（配置 `single_variety_fast_track=true`）。
- **⚠️ 合约月份约定**：单品种分析须在品种代码**内嵌合约月份**（如 `LH2609` 而非 `LH`）。`scan_all` 现自动解析 `LH2609 → variety=LH + contract=2609`，透传 `get_kline(contract='2609')` 取**真实合约** K 线（MA60 与文华财经一致）。不带月份（如 `LH`）则取**主力连续 L8**，仅用于全品种筛选，不可用于单品种定合约的 MA 计算。
- **自进化前置**：仅当存在未验证裁决 / 达样本阈值(≥5)时才跑 validate/calibrate/evolve；ML 训练需 feedback≥50。无触发则跳过（本次即如此，秒级）。
- **Agent 看门狗（watchdog）**：任一后台研究员/辩论/裁决 Agent 超过 `agent_watchdog_seconds=420` 秒未落盘产物 → 主管直驱计算/合成（如观澜卡死时主管用已验证适配器+numpy/pandas 算技术面并原子写入），不阻塞整条链路。
- **⚠️ 强制看门狗（根治 Agent 卡死）**：spawn 任意辩论子 Agent 时**必须**遵守 `memory/agent_watchdog.md`：① prompt 显式禁止 WebSearch/WebFetch（链证源 SKILL 默认要求联网验证，是重跑卡死根因）；② 设 `max_turns=10~15` 有界；③ 轮询产物 `poll_file_ready(timeout=420)`，超时 `TaskStop` + 主管直驱。**已实证**：链证源禁联网+有界 spawn 秒级返回（此前因 WebSearch 阻塞卡 11min）。
- **并行最大化**：观澜/探源并行；证真/慎思并行；链证源可与数技源并行。

## 三、预期收益
单品种分析从"十几分钟~半小时"降至"1-2 分钟级"（依赖已冻结 + 看门狗兜底 + 跳过无关扫描）。

## 四、已知数据质量事项
- **ma60 数据质量 bug（已修复 ✅ · 2026-07-07 18:33）**：根因 `scan_all`/`get_kline` 不传合约时默认取**主力连续合约(L8)**，MA60 与真实合约不符（通达信L8=11279.5 / AKShare错误=11372，均≠文华真实合约LH2609=**12188.67**）。修复：`scan_all.collect_kline_for_all` 新增 `_split_symbol_contract()`，`LH2609` 自动解析合约月份并取真实合约 K 线。验证：`scan_all -s LH2609` → ma60=**12188.67**（与文华一致），price=12285 仅高于 ma60 +0.79%（几乎贴线），MA20(12035)<MA60(12188) 空头排列初期。
- **改进**：单品种取数强制带合约月份（如 LH2609）走真实合约；全品种筛选仍用连续合约。
