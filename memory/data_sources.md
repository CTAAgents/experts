# 数据源规则

## 🔴 链证源规则
- 60日滚动Pearson相关系数(r>0.80)检测冗余品种，一链一代表
- 数据溯源：价格/技术数据来自数技源，基本面来自WebSearch（附URL+日期）
- 不下多空结论，只描述事实

## 🔴 数据源路由
- 国内期货数据统一由 `futures-data-search` 调度
- **盘中子周期**降级链: TDX → TqSDK(R0归一化) → AKShare分钟 → 东方财富 → AKShare日线
- **盘后子周期**降级链: TDX → AKShare分钟 → 东方财富 → TqSDK(R0归一化) → AKShare日线
- **日线/周线/月线**: TDX → TqSDK → 东方财富 → AKShare日线
- 辩论专家团记忆写入自身 `memory/` 目录，不写入宿主工作空间

## 🔴 R0 子周期K线归一化基准（2026-07-10 掌柜确立·最高优先级）

**时间周期切片以通达信、文华财经、博易大师等主流期货软件的划分方法为准。**

- 主流软件 = 交易所会话感知切分（详见 `memory/session_rules.md`）
- 任何数据源若切片方式与主流软件不一致，**必须在数据适配层转换为会话感知格式后方可使用**
- 转换方式: 检测bar间时间间隔，gap>120min识别会话边界，按会话重新聚合
- 已实施: TqSDK子周期命中后强制`normalize_sub_period_bars()`归一化

## 🔴 R26 所有OHLCV消费者必须遵守会话划分（2026-07-10 掌柜确立）

**凡涉及子周期OHLCV数据的技术分析、指标计算、回测、策略评分——均须使用会话感知K线。**

- 数据入口: `multi_source_adapter.get_kline(period=60m/120m/240m)` → 已内置R0归一化
- 策略层: channel_breakout / layered_l1l4 / factor_timing / three_signal — 均从tech dict读取预计算指标 ✓
- 技术分析skill: `data_interface.py` 通过multi_source_adapter间接访问 ✓
- 回测/优化器: `backtest/` `optimizer/` 使用预采集数据 ✓
- **禁止**: 任何模块直接调用akshare/tqsdk/tdx原始接口获取子周期OHLCV后直接使用
- **例外**: `indicators/calc_core.py` 为纯函数公式库，不涉及数据访问，豁免

## 🔴 子周期技术指标数据源规则（2026-07-08 掌柜确立·R19-R23）

- 子周期(60m/120m/240m)技术指标优先以**TDX TQ-Local**为数据源
- TDX不可用时，降级数据（AKShare分钟/东方财富）必须在报告中标注"数据源: AKShare分钟+合约代号"
- **R23 拒绝原则**：如果所有数据源均不可用（TDX过期→AKShare分钟挂起→东方财富不通→AKShare日线无子周期），明鉴秋必须**直接拒绝分析**，退出流程，告知用户每个源的失败原因
- RSI/CCI等短期指标降级可用；ADX等全序列Wilder平滑指标必须标注显著差异风险
- 不同连续合约(SC0 vs L8等)可能导致ADX偏差>100%
- **R24 价格真实性**：分析所用最新价必须为真实成交价。盘中=采集时刻最新价；盘后=末笔收盘价；休盘期间=休盘前最后成交价。严禁杜撰/过期/模拟数据
- **🆕 R25 TqSDK子周期归一化** (2026-07-10): TqSDK子周期K线使用纯时钟窗口(7200秒), 不识别交易时段边界。盘中优先使用(实时性)，命中后强制走`normalize_sub_period_bars()`转换为会话感知格式。盘后作为兜底同样归一化。

## 🔴 子周期K线会话划分规则 (2026-07-10 确立)

详见 `memory/session_rules.md`。核心:
- 子周期K线须按交易所交易时段划分，非简单自然钟均分
- 跨会话检测: 相邻bar时间差>120min → 新会话
- `resample_60m_to_120m` 使用会话感知合并，不跨夜盘/午休边界
- 各品种120m bar数/日因夜盘长度而异(2~5根)

## 定性信息门户目录

详见 `memory/info_portals.md`。该目录**不参与本表 A/B/C/D 定量评级体系**，原因如下：

- **性质隔离**：定量 feed（价格/持仓/库存等）可由程序自动拉取，需可靠性评分校准置信度。定性门户（政策/研报/产业动态/现货基差）为人工查阅源，不参与自动化置信度折扣。
- **置信度统一**：定性门户来源的证据在辩论中置信度统一按 1.0 处理（同团队数据架构约定），不因门户本身的权威层级打折。
- **使用指引**：明鉴秋在补充定性证据时优先查阅 `info_portals.md` 所列站点，各研究员（探源/观澜/链证源）不做 WebSearch/WebFetch，由协调员统一调度。

## 风控明发现记录

### 2026-07-13 AO vol_ratio 数据分歧
- **发现**：AO品种成交量比率(vol_ratio)在不同数据源/时间点存在显著分歧
  - 技术分析skill(09:26 通达信日线) → vol_ratio=0.08（极低，均量的10-15%）
  - quant-daily scan(晚间) → vol_ratio=1.08（正常，略高于均量）
  - 差异达13.5倍，根本性地影响成交量风险评估结论
- **根因推测**：①日线vs分钟线口径差异 ②不同采集时间点的快照差异 ③数据源降级链造成统计周期不一致
- **建议**：AO品种后续分析需统一标注vol_ratio计算口径（日线/分钟线/快照时间点），避免同一指标得出相反结论
