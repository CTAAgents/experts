## 2026-07-06 | ADX角色重定义：趋势过强警示，不做开仓依据（v1.0）

**触发来源**: 掌柜在系统重构中明确ADX的角色定位。
**问题诊断**: 旧系统将ADX<15作为排除开仓的条件、ADX>60作为强信号推荐，这两个方向都错了。突破初期ADX天然低位，行情末端ADX最高。
**影响范围**: 全系统所有依赖ADX做决策的环节（闫判官辩论筛选、风控明仓位管理、观澜技术分析、策执远策略制定、phase3报告分级）。

### 修正规则

| 规则编号 | 规则内容 | 类型 | 优先级 |
|:--------|:--------|:----|:------|
| R11 | ADX<15 **不作为**不开仓依据 | 硬约束 | P0 |
| R12 | launch阶段+ADX<25：属于正常突破特征，鼓励早期参与，不做额外限制 | 硬约束 | P0 |
| R13 | ADX>50：提示"趋势已运行较远"，仓位上限减半 | 硬约束 | P0 |
| R14 | ADX>60：标记🔴警示"追高/追空风险高"，建议放弃或仓位≤3%。**但V型反转时不适用此条风险警示** | 硬约束 | P0 |
| R15 | 辩论品种筛选不依赖ADX绝对值，以signal_type（breakout/pullback/gap）为准 | 硬约束 | P0 |
| R16 | V型反转识别：价格在短时间内（≤5根K线）完成方向逆转，穿越MA20/MA60，幅度≥5×ATR。此时ADX可能仍处于高位，但不适用ADX>60风险警示——因为价格已不在追高/追空方向，而是处于反转后的初期阶段 | 硬约束 | P0 |
| R17 | V型反转时的ADX处理：即使ADX>60，也不标记追高追空风险。改用阶段=reversal判断代替ADX判断 | 硬约束 | P0 |
| R18 | Agent（风控明/闫判官/策执远）在读取ADX>60规则时，必须**先检查是否为V型反转**。若是则跳过ADX风险警示，按反转处理 | 硬约束 | P0 |

**触发来源**: 用户在第八次全品种辩论后质疑Top5推荐全部RSI<37，PX+PR同链集中。
**问题诊断**: 纯ADX×共振评分忽略RSI极端值和入场时机，且链内去重只在下游执行，未在Top5推荐阶段执行。
**影响范围**: SELL高置信度品种全部集中在RSI 29-36区域（l/PX/PR/eg/MA），追空性价比低。

### 修正规则

| 规则编号 | 规则内容 | 类型 | 优先级 |
|:--------|:--------|:----|:------|
| R01 | 空头方向 RSI < 30 → 禁止标记为SELL高，强制降级至SELL中 | 硬约束 | P0 |
| R02 | 空头方向 RSI < 35 → SELL高需标注 ⚠️ 超卖风险标签 | 软约束 | P1 |
| R03 | Top5推荐中，同产业链最多保留1个品种（评分最高者） | 硬约束 | P0 |
| R04 | Top5推荐必须覆盖 ≥5 条不同产业链 | 硬约束 | P1 |
| R05 | 最终排序引入安全边际因子: `composite = ADX_adj × (1 + safety/30)`，其中 `safety = max(0, |RSI-30|)`, `ADX_adj = ADX(无冲突) 或 ADX×0.6(有冲突)` | 评分修正 | P1 |

### 验证案例（2026-07-06 实战）

- **MA甲醇** (RSI=29.2, ADX=80.2): 按R01降为SELL中，虽有全品种最高ADX但不进入Top5首选
- **PX/PR**: 按R03同一聚酯链只保留PR(ADX=76.2)，PX被链去重剔除出Top5
- **l塑料** (RSI=30.0): 踩在超卖线上，按R02标记⚠️超卖风险
- **SM锰硅** (RSI=47.0): 按R05安全边际=17.0跳升，从旧排名第9升至修正后第2

### 后续验证计划

- 未来5次辩论中统计：应用R01-R05后Top5推荐的市场表现 vs 旧算法Top5
- 若修正后胜率下降，需回滚并重新校准R05权重系数
- 若某条规则连续3次触发但价格走势未发生对应修正 → 该规则标记为过度保守

## 2026-07-06 | 数据质量与确认偏误修正（v2.0）🆕

**触发来源**: 用户在生猪(LH)单品种辩论后逐一指出4处数据引用错误，裁决结论建立在错误/过时数据上。
**问题诊断**: 
1. 确认偏误：辩论框架预设空方胜→不自觉筛选支持空头证据→忽略反向数据
2. 数据时效：WebSearch结果未检查文章日期，5天前文章当"当前"
3. 逻辑倒置：高屠宰量=去库存≠供应过剩，因果方向审查缺失
4. 异常值：系统已过滤的35.86%斜率仍被当作"深度Contango"论据
5. 源不透明：5个网页源未标注出处
**影响范围**: 整个辩论流程（研究员→辩手→闫判官→明鉴秋），无Agent幸免。

### 修正规则

| 规则编号 | 规则内容 | 类型 | 优先级 | 适用Agent |
|:--------|:--------|:----|:------|:--------|
| R06 | **数据时效性检查**：WebSearch结果必须检查日期，>3天标注"可能已过时"。引用非当日K线须标注"截至YYYY-MM-DD" | 硬约束 | P0 | 研究员、辩手、闫判官 |
| R07 | **反向证据强制检索**：裁决前搜索与裁决方向相反的最新数据。反向证据强度≥正向50%→触发rematch。裁决输出含"主要反向风险"段落 | 硬约束 | P0 | 闫判官 |
| R08 | **逻辑方向核验**：每个论据必须通过因果方向测试。数据变化→供需影响→价格方向。例：屠宰量↑→去库存↑→远期供应↓→价格↑ | 硬约束 | P0 | 所有辩手 |
| R09 | **异常值引用禁令**：系统标记"异常"/"过滤"/"unknown"的值禁止作论据。如需引用须注明+提供独立验证 | 硬约束 | P0 | 所有Agent |
| R10 | **数据源标注强制**：每条外部数据标注来源URL+采集日期。内部数据标注采集器+K线截止日期 | 硬约束 | P1 | 明鉴秋 |

### 验证案例（2026-07-06 LH辩论）

- **LH2607数据**: 引用7月1日"-2.62%暴跌"→实际7月6日+5.96%（数据过期5天+方向完全相反）
- **Q1屠宰量**: +18.1%被当作"供应过剩"利空→实际Q1屠宰高=已去库存=Q2后供应压力减轻（因果倒置）
- **远月Contango**: 系统标记"异常已过滤"→仍作"深度Contango"论据引用（违规R09）
- **数据源**: 雪球/搜猪网/新浪/B站/百家号5个源均未标注（违规R10）

### 后续验证计划

- 每次辩论裁决后，明鉴秋逐条检查R06-R10触发情况
- 任一条触发→裁决标注"⚠️数据质量风险"，风控评级升一级
- 连续3次无触发→R06-R10标记为"已内化"，降级为P2提示
