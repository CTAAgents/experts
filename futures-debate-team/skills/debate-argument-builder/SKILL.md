# 通用期货论点构建器

## 依赖
- **输入方**：`BullOutput` / `BearOutput`（`contracts/debate.py`）
- **上游依赖**：依赖 P1 `DataCollectionOutput`（data）、P1 `TechnicalOutput`（tech）、P2 `ChainAnalysisOutput`（chain）
- **版本**：`2.1`（所有 schema 带 `version` 字段，编排层 `parse_and_migrate` 自动路由）
- **输出方式**：正文（Markdown 人类可读）+ 末尾 ```json fence 结构化摘要

## 概述

**双模式设计**：

- **独立使用模式（单人）**：LLM推理 + 主动基本面搜索。自行使用 WebSearch/WebFetch 收集数据。
- **辩论专家团集成模式（团队）**：基于研究员快照提炼论据。**辩手不做独立WebSearch**，所有基本面数据由基本面研究员（探源）和技术面研究员（观澜）预先搜集并提供。

**可复用于**：期货交易辩论专家团、自定义分析管道、研究报告生成、交易日志分析等。

## 角色模式

| 模式 | 方向 | 输出 |
|:----:|:----:|------|
| **bull** | 🟢 多头 | 做多看涨论点 |
| **bear** | 🔴 空头 | 做空看跌论点 |

## 独立使用模式

### 输入格式

接受任意品种的结构化市场数据：

```json
{
  "symbols": [
    {
      "id": "品种代码",
      "direction": "BUY/SELL",
      "price": 当前价格,
      "technical": {
        "trend_stage": "启动/主升/主跌/衰竭/反转",
        "ma_alignment": "多头/空头/交叉/震荡",
        "adx": 数值,
        "rsi": 数值
      },
      "volume_price": {
        "oi_trend": "增/减/稳",
        "price_volume": "价涨量增/价涨量缩/价跌量增/价跌量缩",
        "basis": "走强/走弱/平稳"
      },
      "term_structure": "contango/back/flat",
      "chain": {
        "name": "产业链名",
        "consistency": 0-100
      },
      "data_quality": "正常/降级/缺失"
    }
  ]
}
```

### 输出格式

采用**双轨输出**格式：

**① 正文（人类可读）**：用连贯 Markdown 正文撰写，6 维度分析框架，附具体数字和引用来源。

**② 结构化摘要（```json fence 机器消费）**：在正文末尾另起一行，用 ```json fence 输出以下字段，字段名不可改：

```json
{
  "variant": "bull",                          // bull 或 bear，固定值
  "dimensions": [
    {"dim": "供给", "claim": "核心观点", "evidence": "仓单环比-12%", "confidence": 0.8},
    {"dim": "需求", "claim": "核心观点", "evidence": "同比+5%", "confidence": 0.6}
  ],
  "summary_4_risk": "供给收缩+基差走强，主要风险在...",  // ≤100字，给风控的精简版
  "full_text": "见上方正文",                              // 固定值，与正文一致
  "confidence": 0.76,                                     // 整体置信度0-1
  "rebuttal_targets": []                                   // 本轮反驳了熊的哪些维度，空数组表示首轮
}
```

**解析方式**：下游从 ```json fence 中 parse 结构化数据，正文保留给 HTML 报告/人类阅读。

在正文末尾另起一行，用 ```json fence 输出以下字段，字段名不可改：

  "confidence": 0.76,                                     // 整体置信度0-1

    "bull_case": {"target": 4200, "probability": "20%", "desc": "乐观情景：限产超预期"},

### Pydantic Schema 定义

当本 skill 被辩论专家团集成使用时，按以下 schema 验证：

```python
from pydantic import BaseModel
from typing import Literal

class DimensionItem(BaseModel):
    """单维度分析项"""
    dim: str                      # 维度名称
    claim: str                    # 核心观点
    evidence: str                 # 可核验证据（必含具体数字，格式：`"数值（来源：XXX，截至X月X日）"`）
    confidence: float             # 该维度置信度 0-1

class AffirmativeSchema(BaseModel):
    """证真（正方辩手）论点。variant="affirmative"：论证数技师方向的正确性"""
    variant: Literal["affirmative", "opposition"]
    dimensions: list[DimensionItem]
    summary_4_risk: str           # ≤100字，给风控的精简摘要
    full_text: str                # "见上方正文"
    confidence: float             # 整体置信度 0-1
    rebuttal_targets: list[str]   # 本轮反驳了对手的哪些维度名，首轮为[]
```

**产出规范（双轨输出）**：
- Agent 产出同一份消息包含：① Markdown 正文（人类可读，给 HTML 报告） ② 末尾 ```json fence 结构化摘要（给明鉴秋机器消费）
- 明鉴秋从 ```json fence 中 parse 结构化数据，正文保留给 HTML 报告
- 下游通过 `obj.dimensions`、`obj.summary_4_risk` 等属性访问
- `full_text` 固定为"见上方正文"（结构化摘要不重复正文，正文在 fence 外保留）
- 过渡期兼容：如果明鉴秋扒不到 JSON fence，走旧路径将全文当文本传递

## 辩论专家团集成模式

当被 `futures-trading-analysis` 辩论系统的 **正方辩手（证真）** 或 **反方辩手（慎思）** Agent 加载时：

**输入**：由 明鉴秋 传入辩论候选品种的 typed 结构化数据，**包含研究员快照**（`DebateState` 的 `data`/`tech`/`chain`/`research_snapshots` 字段）
**产出**：正文（人类可读）+ 末尾 ```json fence 结构化摘要 → SendMessage → main
（明鉴秋从 fence 扒结构化数据，正文保留给 HTML 报告。过渡期保持双轨，旧路径继续可用。）

**核心原则：辩手不做数据搜集，只做论据提炼。** 所有基本面数据由基本面研究员（探源）和技术面研究员（观澜）在准备期预先搜集并提供。辩手无需也不应该自行使用 WebSearch/WebFetch。

**Prompt 硬约束（L2 辩论铁律）**：
当本 skill 被辩论专家团集成使用时，以下六条硬约束必须嵌入 system prompt：

1. **禁止附和语**：回应对手时 ❌ 禁止用"看多/空方说得有道理""你补充得很好""但是反过来"开头。必须直入主题"熊曰[X维度：证据] → 牛驳：反证"
2. **证据强制**：每个维度 ≥1 个可核验数字（合约价、基差、仓单、持仓）。**所有数据必须来自研究员快照中的内容，不得自行搜索或编造**。没有可引用的具体数字时，该维度标注"数据不足"并降置信度
   - **证据格式强制**：每个 evidence 字段必须包含来源+日期，格式如下：
     ```
     "evidence": "具体数值+单位（来源：XXX机构/媒体，截至X月X日）"
     ```
     示例：`"五大钢材总库存1601.99万吨（来源：Mysteel周度数据，截至6月30日）"`
   - 违反此格式的 evidence 算作不合格，风控明必须将其标注为"糊弄"
3. **角色锚定**：正方辩手（证真）为"信号捍卫者——数技师说多就论证多，说空就论证空"，反方辩手（慎思）为"信号挑战者——站在数技师方向的对立面找漏洞"。双方立场由数据决定，不是预设的多/空。
4. **场景分离**：必须将"基准情景（baseline）"和"乐观/悲观情景（bull/bear case）"分开标注，不得混为一谈
   - `scenarios.baseline`: 最可能的情景（概率>50%）
   - `scenarios.bull_case` / `scenarios.bear_case`: 有利/不利的尾部情景
5. **认错信号（key_validation）**：每个 final proposal 必须附带一个"什么条件下我认错"的触发条件，如"厂库环比连升3周且back翻contango → 多单离场"
6. **🚫 禁止自行搜索**：辩手**不得**使用 WebSearch/WebFetch 自行搜集数据。所有基本面数据必须从研究员提供的研究快照中提取。如果研究员未提供某维度的数据，应标注"研究员未提供该维度数据"并降置信度，不得自行搜索或编造。

**重要 — 全品种覆盖规则**：
- 正方辩手必须对**每个品种**都站在数技师指示的方向上论证
- 反方辩手必须对**每个品种**都站在数技师指示的对立面上质疑
- 正方辩手论证"这个方向为什么对"
- 反方辩手论证"这个方向哪里可能错"

**研究员数据驱动**：在分析过程中，辩手必须从研究员快照中提取以下信息：
- **基本面数据**：探源（基本面研究员）搜集的供需/库存/利润/政策数据
- **技术面数据**：观澜（技术面研究员）提供的量价/关键位/持仓数据
- **产业链数据**：链证源提供的产业链方向/集中度/冗余判断
- 如果研究员快照中缺少某品种某维度的数据，标注"研究员未覆盖"并降置信度

**交叉质询模式**：
- 步1：正方辩手首轮产出 `affirmative_v1`（立论：方向是正确的）
- 步2：反方辩手读 `affirmative_v1` 后产 `opposition_v1`（质疑：方向哪里不对）
- 步3：正方辩手读 `opposition_v1` 后产 `affirmative_v2`（rebuttal，max=1）
- 终止条件：如果正方v2对反方v1的rebuttal里 ≥3/5 维度承认"这点质疑成立，但…" → 可提前结束
- 非对称性：反方只产一次（opposition_v1），第二回合是正方rebuttal（affirmative_v2）

## 工作流程

1. 读取 data + tech + chain + **research_snapshots（研究员快照）** 的数据
2. 逐维度分析（供给、需求、库存、基差、宏观、产业链、市场情绪等）
3. **如果是第二轮（有 rebuttal_targets 输入）**：先写 Rebuttal 段，再写己方更新版。禁止用"你说得对""但是反过来"开头，必须有数据支撑
4. 输出完整正文 + 末尾 ```json fence 结构化摘要
5. ⚠️ **辩手不自行搜索**：所有数据必须来自研究员快照，不可使用 WebSearch/WebFetch

| 维度 | 多头（做多） | 空头（做空） |
|------|-------------|-------------|
| **1. 趋势结构** | 通道突破上轨、MA多头排列、MACD金叉、Higher High | 通道跌破下轨、MA空头排列、MACD死叉、Lower Low |
| **2. 量价关系** | OI增+价涨→健康；基差走强→现货支撑 | OI减+价跌→溃散；基差走弱→现货疲软 |
| **3. 期限结构** | Back→做多有利，近月溢价 | Contango→做空有利，展仓收益 |
| **4. 产业链验证** | 方向一致→加强；背离→需合理解释 | 同左（对称） |
| **5. 基本面/市场情绪** | **独立模式**：通过WebSearch/WebFetch搜索当前新闻、供需报告、政策动向、库存数据<br>**辩论团模式**：从研究员快照中提取基本面/市场情绪数据 | 同左（对称） |
| **6. 风险点** | 至少列出2个风险 | 至少列出2个风险 |

### 维度权重（隐性·影响最终置信度）

虽然没有显式权重字段，但各维度对最终置信度的影响程度不同：

| 维度 | 对总分影响 | 原因 |
|:----|:---------:|:-----|
| **1. 趋势结构** | 30% | 价格行为是核心，右侧交易优先 |
| **2. 量价关系** | 20% | 量价配合提供二次确认 |
| **3. 期限结构** | 15% | Back/Contango提供结构方向 |
| **4. 产业链验证** | 15% | 产业传导提供辅助验证 |
| **5. 基本面/市场情绪** | 10% | 滞后指标，仅做一致性检查 |
| **6. 风险点** | 10% | 风险控制，不影响方向判断 |

最终置信度 = Σ(维度置信度 × 维度权重)。结果再与数技师方向置信度进行综合。

### 数技师方向置信度挂钩规则（新增）

辩手的整体confidence必须与数技源信号强度挂钩：

| 数技师信号等级 | 辩手confidence上限 | 示例 |
|:--------------|:-----------------:|:-----|
| STRONG (±70~100) | ≤0.85 | rb(-76)→正方conf≤0.85 |
| WATCH (±50~69) | ≤0.75 | c(-74)→正方conf≤0.75 |
| WEAK (±30~49) | ≤0.65 | PK(+59)→正方conf≤0.65 |
| NOISE (±0~29) | ≤0.50 | 不开展正式辩论 |

**规则**：辩手最终confidence = min(自然推理得出的置信度, 数技师等级对应的上限)。违反此规则的输出标注"信号等级与置信度不匹配"并降级。

## ⚠️ 数据来源铁律（辩论专家团专用）

**本铁律仅适用于辩论专家团集成模式。独立使用模式请参考独立模式的搜索指引。**

**辩手的所有论据数据，来源只有一条：研究员快照。** 辩手不得自行搜索、不得编造、不得推测。

1. **研究员快照是唯一来源**：所有基本面数据必须来自探源（基本面研究员）和观澜（技术面研究员）提供的研究快照。辩手在快照范围之外不得自行寻找或引入数据。
2. **引用可追溯**：证据中引用的数据，必须同时注明"来自X研究员提供的快照"和原始来源。
3. **研究员未覆盖的标示**：如果研究员快照中缺少某品种某个维度的数据，辩手必须标注"研究员未覆盖该维度数据"并降置信度。
4. **禁止编造**：**严禁编造模拟数据、虚构新闻报道、伪造官方公告、生成虚假库存/产量数字。** 研究员未提供的数据，不得自己编。
5. **格式强制**：每个 evidence 字段必须包含来源+日期，格式：`"数值+单位（来源：XXX，截至X月X日——数据来自探源快照）"`

## 🔒 Anti-Hallucination Circuit Breaker（新增·全局强制）

**以下约束在所有模式下强制执行，违反即输出降级或终止。**

### 1. 置信度上限约束
| 字段 | 上限 | 原因 |
|:----|:----|:-----|
| `dimensions[].confidence` | **≤0.90** | LLM对单维度的置信度不可超过90%，超过即视为幻觉 |
| 顶层 `confidence` | **≤0.85** | 整体置信度比单维度更低——多个维度综合后不确定性叠加 |
| `scenarios[].probability` | 总和=100% | 三种情景概率必须精确等于100%，否则格式不合格 |

**执行方式**：输出JSON后风控明自动校验。违反任意一条confidence上限标注为"hullucination_risk"并降级。

### 2. Token Guard（输出长度限制）
| 维度 | 上限 | 超限处理 |
|:----|:----|:---------|
| 单条 evidence 长度 | **≤200字符** | 超限截断至200字符，末尾加"…(截断)" |
| summary_4_risk | **≤100字** | 超限截断（格式强制） |
| 单维度 claim | **≤50字** | 超限截断 |
| 全文输出 | **≤8000 tokens** | 超出则自动结束rebuttal轮，不再生成v2 |

### 3. Rebuttal Circuit Breaker
| 防呆机制 | 规则 |
|:---------|:-----|
| rebuttal_targets 去重 | 同一维度名在targets中出现≤1次（重复自动去重） |
| rebuttal_targets 上限 | **最多5个**，超出只取前5个 |
| rebuttal 轮次上限 | **max=1**（已存在）——确认为硬限制，不可扩展 |
| 维度数量上限 | 每方**3-5个**维度，超出自动裁剪 |

### 4. 重复内容检测
- rebuttal v2与v1的重复率 >30% → 本轮作废，直接跳过
- 检测方式：计算v2全文与v1全文的Jaccard相似度

### 5. 递归/循环终止
- 交叉质询最大深度：**2层**（v1→v2→终止）
- 禁止自行调用自身（不允许自我递归）
- 如果连续3次输出被标记为"hullucination_risk" → 触发硬终止，输出降级通知

## ⏱️ 资料时效性铁律（辩论专家团专用）

**本铁律仅适用于辩论专家团集成模式。**

辩手不自行判断数据时效性——**时效性由研究员在提供快照时标注**。辩手应参考研究员标注的时效性来决定置信度：

| 研究员标注 | 辩手处理 |
