## 改动一：论据分类化（F1-F5族标注）

### 目标

让正反方辩手的每条论据都自带一个"论证策略族"标签，使后续的加权裁决和统计分析成为可能。

### 改动文件

- `agents/futures-affirmative-debater.md` — 正方辩手提示词
- `agents/futures-opposition-debater.md` — 反方辩手提示词

### F1-F5 论证策略族定义

在规则的 `constraints` 区域或辩论素材包中嵌入以下分类系统：

```
## 📐 论证策略族分类系统（FDT-F1-F5）

每条论据必须标注所属的策略族类别。分类系统如下：

| 族代码 | 名称 | 核心思想 | 典型论据示例 |
|-------|------|---------|-------------|
| F1 | 技术面量价分析 | 从价格和成交量中提取方向信号 | "MA5上穿MA20形成金叉"、"ADX=45强趋势"、"BB带宽扩张+带量突破" |
| F2 | 基本面供需分析 | 从供需矛盾、库存变化、基差结构中判断 | "社会库存同比-15%"、"基差走强+近月升水"、"开工率低于同期" |
| F3 | 持仓资金分析 | 从资金流向和主力持仓中判断 | "主力席位多头增仓+20%"、"持仓量创新高"、"牛现减多分歧" |
| F4 | 宏观政策驱动 | 从宏观政策、地缘事件中判断 | "FOMC降息预期升温"、"OPEC+减产决议"、"出口退税调整" |
| F5 | 套利结构分析 | 从跨期/跨品种价差结构挖掘 | "展期收益为正，做多近月"、"卷螺差处于历史低位" |

### 论据格式要求

每个论点 (CLAIM) 输出格式扩展为：

```
论点ID: 证真-D1
策略族: F1  ← 🆕 新增字段
CLAIM:  螺纹钢日线MA5上穿MA20，多头信号确立
EVIDENCE: MA5=3080, MA20=3055, 穿越幅度+0.8%, 2026-07-09数据
REASONING: 均线金叉→短期趋势转多→多头动能增强
IMPACT: HIGH
```

### 多条论据覆盖多族的加分规则

- 如果本方3-5条论据覆盖了 ≥3个策略族 → 证据维度评分 +0.5
- 如果本方论据全部集中在1个族 → 证据维度评分 -0.5（警示"证据来源单一"）
- 在 final statement 中，必须总结"本方论据覆盖了X个策略维度"
```

### 更新脚本

新建 `scripts/update_matrix.py`（用现有 `scripts/memory_writer.py` 的更新机制调用）：

```python
"""品种×策略族适应性矩阵更新脚本。
 
每次闫判官裁决 + 实际行情验证后调用。
TODO: 与实际的行情验证流程对接后，将硬编码替换为真实数据。
"""

import json
import os
from datetime import datetime

MATRIX_PATH = os.path.join(os.path.dirname(__file__), "..", "memory",
                           "instrument_strategy_matrix.json")
LEARNING_RATE = 0.3
MIN_SAMPLES = 3


def load_matrix():
    with open(MATRIX_PATH) as f:
        return json.load(f)


def save_matrix(matrix):
    with open(MATRIX_PATH, "w") as f:
        json.dump(matrix, f, indent=2, ensure_ascii=False)


def update_family_weight(matrix, symbol, family_code, was_correct):
    """
    更新单一策略族在特定品种上的权重。
    
    Args:
        matrix: 矩阵数据
        symbol: 品种代码 e.g. "rb"
        family_code: 策略族代码 "F1"-"F5"
        was_correct: 该策略族的论据方向是否被后市验证正确
    """
    entry = matrix["data"].get(symbol)
    if not entry:
        return
    
    fam = entry["families"].get(family_code)
    if not fam:
        return
    
    fam["v"] += 1
    lr = matrix["meta"]["learning_rate"]
    # EMA在线更新: w_new = w_old + lr * (was_correct - w_old)
    fam["w"] = round(fam["w"] + lr * (1.0 if was_correct else 0.0 - fam["w"]), 3)
    fam["updated"] = datetime.now().strftime("%Y-%m-%d")
    
    # 钳制到 [0.05, 0.95]
    fam["w"] = max(0.05, min(0.95, fam["w"]))


def batch_update(symbol, family_results):
    """
    批量更新一个品种的多族权重。
    
    Args:
        symbol: 品种代码
        family_results: dict of {family_code: was_correct}
    """
    matrix = load_matrix()
    for family_code, was_correct in family_results.items():
        update_family_weight(matrix, symbol, family_code, was_correct)
    save_matrix(matrix)


if __name__ == "__main__":
    # 示例用法
    batch_update("rb", {"F1": True, "F2": True, "F3": False})
    print("矩阵已更新")
```

## 改动三：闫判官加权裁决

### 目标

闫判官的裁决从"综合判断"升级为**按品种适应性矩阵加权汇总论据**的系统性评估。

### 改动文件

- `agents/futures-judge.md` — 闫判官提示词（评分模型 + 判决逻辑）

### 评分模型升级

将现有评分模型从 6 维等权替换为**分层加权评分模型**：

#### 替换前（现有）

```
| 维度 | 权重 | 评分标准（1-10） |
|:----|:----:|:----------------|
| 逻辑严谨度 | 25% | 论证有无断层... |
| 证据充分性 | 20% | 是否引用通道突破信号数据... |
| 量化一致性 | 15% | 是否与所引用的策略数据吻合... |
| 反驳有效性 | 20% | 是否正面回应对方核心论点... |
| 风险意识 | 10% | 是否给出合理止损/仓位... |
| 表达与结构 | 10% | 论点清晰... |
```

#### 替换后

保留原有的 6 维评分结构不变，在评分前增加一个**族加权预处理步骤**：

```markdown
### 🆕 族加权预处理（基于 F1-F5 × 品种适应性矩阵）

在进入评分之前，先对正反双方的论据进行族加权处理：

```
步骤 1: 收集双方论据列表，提取每条论据的 [策略族(F1-F5), IMPACT(H/M/L)]
步骤 2: 读取 memory/instrument_strategy_matrix.json 获取当前品种的各族权重
步骤 3: 计算加权有效论据得分（WEAS, Weighted Effective Argument Score）

WEAS(正方) = Σ IMPACT_numeric(论据i) × w(族j, 品种)
WEAS(反方) = Σ IMPACT_numeric(论据j) × w(族k, 品种)

IMPACT映射: HIGH=3.0, MEDIUM=1.5, LOW=0.5
w(族, 品种) 来自 matrix.json

步骤 4: 族多样性检查
- 若一方论据覆盖 ≥3个族 → 证据充分性 +1分
- 若一方论据覆盖 ≤1个族 → 证据充分性 -1分
```

**族加权预处理不直接决定胜负**，而是：
- 为"证据充分性"维度提供差异化初始分值
- 为后续"量化一致性"评分提供"论据是否正确定位到对应策略族"的检查依据
- 最终的裁决仍然由 6 维评分模型加权决定，但各维度的输入包含族加权的信息
```

```markdown
## 评分模型（通道突破信号+多因子视角+族加权）

### 🆕 步骤一：族加权预处理

在进入 6 维评分之前，先执行族加权预处理：

```
1. 收集双方论据列表（正反方各3-5条），提取每条论据的 [策略族(F1-F5), IMPACT]
2. 加载 memory/instrument_strategy_matrix.json → 获取当前品种的各族权重 w
3. 计算 WEAS（Weighted Effective Argument Score）:
   WEAS(正方) = Σ IMPACT_numeric(论据i) × w(族j)
   WEAS(反方) = Σ IMPACT_numeric(论据j) × w(族k)
   IMPACT映射: HIGH=3.0, MEDIUM=1.5, LOW=0.5
4. 族多样性检查:
   - 覆盖 ≥3个族 → 证据充分性 +1分
   - 覆盖 ≤1个族 → 证据充分性 -1分

   - 策略族标注错误或缺失 → 量化一致性 -1分
```

> WEAS 不直接决定胜负，而是为证据充分性和量化一致性两个维度提供差异化初始参考。
> 族覆盖率越高、标注越准确，这两个维度的初始分越高。

### 步骤二：六维加权评分

| 维度 | 权重 | 评分标准（1-10） | 族加权影响 |
|:----|:----:|:----------------|:-----------|

| **证据充分性** | 20% | 是否引用通道突破信号数据+辅助验证数据、数据口径正确、非纯叙事 | 🆕 族多样性调整: ≥3族+1分, ≤1族-1分 |
| **量化一致性** | 15% | 辩手观点是否与所引用的策略数据吻合；能否合理解释分歧；**论据的策略族标注是否准确** | 🆕 族标注准确性: 标注正确+0.5分, 错误/缺失-1分 |
| **反驳有效性** | 20% | 是否正面回应对方核心论点、有无遗漏 | — |
| **风险意识** | 10% | 是否给出合理止损/仓位、区分基准与尾部情景 | — |
| **表达与结构** | 10% | 论点清晰、层次分明、不超时 | — |

> 💡 总分=Σ(维度分×权重)。高分者胜，分差<5分可判draw。
```

## 三改联合效果

| 改动 | 直接影响 | 间接影响 |
|------|---------|---------|
| **论据分类化** | 正反方输出含族标签 | 为后续自动分析和统计提供基础 |
| **品种适应性矩阵** | 闫判官可查询(品种×族)的历史权重 | 矩阵自动更新后，裁决越来越精准 |
| **闫判官加权裁决** | 族多样性和品种权重影响评分 | 裁决理由含 WEAS 摘要，可追溯性提升 |

| 项目 | 改动文件数 | 新增行数 | 风险等级 |
|------|-----------|---------|---------|
