## 四阶段独立化路线

### Phase 1：行为独立（预计1~2周）

> 目标：不依赖平台自动化和外部触发器，专家团自主决策"什么时候做什么"

| 任务 | 当前限制 | 解决方案 |
|:-----|:--------|:--------|
| 日常辩论自调度 | 平台cron: `automation-1782228336539` | 专家团内建心跳机制，`scheduler/` 模块 |
| 条件触发 | 外部脚本检查条件 | 内置条件引擎：时间+数据量+事件三种触发器 |
| 静默模式 | 必须有人与平台交互才能执行 | 后台守护进程，无需前台会话 |
| 状态持久化 | 依赖会话上下文 | 全量状态写入 `memory/`，下次恢复 |

**衡量标准**：平台删除所有automation后，专家团仍能按预设节奏自主运行。

### Phase 2：运行时独立（预计1个月）

> 目标：不依赖平台spawn，专家团作为一个独立进程启动

| 任务 | 当前限制 | 解决方案 |
|:-----|:--------|:--------|
| Agent派生 | 平台`Agent`工具 | 内建Agent注册表 + in-process线程池 |
| 角色间通信 | 平台`SendMessage`协议 | 内建消息队列（内存queue→持久化） |
| 上下文管理 | 平台会话上下文 | 自管理session_id + memory/快照 |
| 自启动 | 用户必须手动选专家团 | `bootstrap.py` 一键启动整个团队 |

**衡量标准**：`python bootstrap.py` 即可完整启动10Agent系统，无需平台介入。

### Phase 3：数据独立（预计1~2个月）

> 目标：不依赖平台skill，数据采集+计算+存储全在专家团内部

| 任务 | 当前限制 | 解决方案 |
|:-----|:--------|:--------|
| TDX连接 | 平台TDX connector / TQ-Local | 内建 `data/connectors/tdx_standalone.py` |
| 东方财富 | 平台WebFetch / 东方财富skill | 直接API调用（requests） |
| 指标计算 | quant-daily skill（平台管理） | 内置 `indicators/calc_core.py`（已存在） |
| 因子计算 | factor_timing strategy（平台skill） | 内建 `factor_engine/` |
| 德比判决 | futures-trading-analysis skill | 内建 `debate_engine/` |

**衡量标准**：专家团目录独立完整，任何平台skill消失后仍能正常运行。

### Phase 4：部署独立（预计3个月）

> 目标：专家团可作为独立微服务部署，通过API对外提供服务

| 任务 | 解决方案 |
|:-----|:--------|
| API接口 | FastAPI 提供REST/gRPC接口 |
| 外部触发 | webhook回调（如收到新K线自动触发辩论） |
| 多实例 | 不同品种池分配到不同实例 |
| 监控 | 内建prometheus指标 + 告警 |
| 报告推送 | 邮件/飞书/钉钉/Webhook 多通道 |

**衡量标准**：`docker-compose up` 即可启动全栈服务，提供HTTP API。

## Phase 1 详细计划

> 优先级最高：先让专家团"不需要平台告诉它该做什么"

### 1.1 内建调度器 `scheduler/`

```
scheduler/
├── __init__.py
├── engine.py       ← 心跳引擎，检查条件+触发任务
├── triggers.py     ← 三种触发器：时间/数据量/事件
└── tasks.py        ← 预注册任务：辩论/验证/校准/进化/ML训练
```

心跳循环：
```
[启动] → 每60秒心跳 → 检查时间触发器 → 检查数据量触发器 → 触发对应任务
```

### 1.2 删除平台automations（逐步推进）

| automation | 替代方案 | 进度 |
|:-----------|:--------|:----|
| `期货交易辩论专家团` (19:15 工作日) | scheduler内置 | ⬜ |
| `自动发布流水线` (23:05 每日) | 独立gitops | ⬜ |
| `主力映射定时更新` (15:30 每日) | 数据层自带 | ⬜ |

### 1.3 后台模式

```bash
# 前台交互模式（当前）
python bootstrap.py --interactive

# 后台守护模式（Phase 1目标）
python bootstrap.py --daemon
# → 启动心跳循环，静默运行，按预设条件触发任务
# → 输出到 scheduler/scheduler.log
```

## 决策原则

1. **依赖向下**：专家团只能依赖基础库（`python stdlib` + `numpy/pandas`），不依赖平台API
2. **记忆自持**：所有状态在 `memory/` 目录内，迁移时目录带走
3. **能力内建**：任何依赖必须显式声明，不能隐式借用平台能力
4. **向前兼容**：独立化过程中保持与当前使用模式兼容（用户仍可对话式使用）
5. **增量替换**：每次替换一个依赖项，验证后再推进下一步
